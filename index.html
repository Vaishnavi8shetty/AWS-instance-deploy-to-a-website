AWS instance deploy to a website

Benefits

1. Scalability: Easily adjust resources to handle traffic spikes.
2. Reliability: High availability with redundant infrastructure.
3. Flexibility: Choose from various instance types and services.
4. Security: Built-in features for protecting data and resources.
5. Global Reach : Serve content from data centers worldwide.

Steps to build the project

Step 1. Launch and EC2 instance

Begin by logging into the AWS Management Console and navigating to the EC2 service. Launch a new EC2 instance by> selecting an appropriate Amazon Machine Image (AMI), choosing the instance type,configuring instance details such as network settings and storage, and finally, creating or selecting a security group to allow inbound HTTP (port 80) and HTTPS (port 443) traffic.

Step 2.Connect to Your Instance

Once your instance is running, connect to it using SSH (Secure Shell) from your local machine. You'll use the private key associated with your chosen key pair to authenticate.

Step 3.Install Required Software

MobaXterm is a comprehensive remote computing tool for Windows that integrates various network tools into a single application. It combines a terminal emulator, X server, SSH client, and other network tools into one package, providing a convenient way to access remote systems and perform various tasks.MobaXterm comes with a built-in SSH client that allows users to securely connect to remote servers and transfer files using the SSH protocol. It supports various authentication methods, including password authentication, public-key authentication, and keyboard interactive authentication. Enter the IP address of the remote host and provide the username specified in the EC2 instance connect section.

Click on "Use private key" and add the key that was created for the instance.Then click ok

Step 5.Upload Website Files

Transfer your website's files (HTML, CSS, JavaScript, images, etc.) to your EC2 instance. You can use SCP (Secure Copy Protocol) or SFTP (SSH File Transfer Protocol) to securely transfer files from your local machine to the instance. Alternatively, you can create a file within the directory /var/www/httpd.

Step 6.Configure Web Server

command -service httpd start

Now, open a new tab and enter the IP address of the instance.

Once everything is set up, test your website to ensure that it's functioning correctly. Check for any errors or issues and make necessary adjustments.
